# 🎮 게이밍 DNA 작업 마스터 지침서 (Gaming DNA Master Instructions)

> **중요**: 이 파일은 게이밍 DNA(포트폴리오 게임 목록) 작업을 수행하기 전 AI가 반드시 가장 먼저 읽어야 하는 최상위 지침서입니다.

## 1. 데이터 관리 원칙 (Data Integrity)
- **Title (명칭)**: 사용자님이 작성한 원본 명칭을 절대 변경하지 마십시오. (대소문자, 띄어쓰기 포함)
- **Company (회사)**: 공식 개발사/유통사 명칭을 유지하되, 누락된 경우에만 보충합니다.
- **Genre (장르)**: 사용자님이 정의한 장르를 유지하며, 시각적 일관성을 위해 필요한 경우에만 표준화합니다.
- **Exclusion (삭제)**: 사용자님이 명시적으로 요청한 항목(예: 포탈 2)만 삭제합니다.

## 2. 이미지 자산 표준 (Asset Standards)
- **파일명 규칙 (Naming Convention)**: 한글/특수문자 인코딩 오류를 방지하기 위해 파일명은 반드시 게임의 **고유 ID**를 기반으로 저장합니다.
    - 예: `id: "console-1"` -> 파일명: `console-1.jpg`
    - 예: `id: "pc-10"` -> 파일명: `pc-10.jpg`
- **이미지 출처**:
    - **스팀/콘솔**: 스팀 공식 상점의 `header.jpg` (대문 이미지)
    - **모바일**: 구글 플레이스토어의 `feature_graphic` (공식 일러스트)
    - **고전 게임**: 게임 시작 타이틀 화면(Title Screen) 또는 공식 키 아트
- **저장 경로**: 모든 이미지는 `public/images/games/` 폴더에 물리적 파일로 존재해야 합니다.

## 3. 작업 워크플로우 (Workflow)
1. **ID 기반 동기화**: 모든 `.ts` 파일의 `image` 경로를 `./images/games/[id].jpg` 형식으로 일괄 수정합니다.
2. **사일런트 다운로드**: 브라우저를 띄우지 않고 백그라운드 스크립트를 사용하여 해당 ID에 맞는 공식 이미지를 수집합니다.
3. **무결성 체크**: 이미지가 0바이트이거나 깨진 경우 즉시 재수집합니다.

## 4. 최종 목록 관리 대상
- `src/data/games/pc.ts`
- `src/data/games/console.ts`
- `src/data/games/mobile.ts`
